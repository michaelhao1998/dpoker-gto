# DPoker GTO 工具需求文档 V1.0
——德州扑克 GTO 快速决策计算器

---

# 1 整体说明

德州扑克新手及进阶玩家在实战牌局中面临决策复杂、时间短暂的困境，传统 GTO 软件（PioSOLVER、GTO+）操作门槛极高，无法在牌桌旁快速使用。本产品旨在打造一款**面向新手友好、快速输入、即时输出 GTO 最优策略**的网页端工具，让玩家能在 60 秒内完成完整策略查询。

**核心设计原则：**
- 最小化输入：可视化交互替代文字键入，减少操作层级
- 近似 GTO：预计算查找表 + 实时简化求解，平衡精度与速度
- 新手友好：策略输出附带简明人话解释

**该部分包括以下模块：**
- 模块 A：牌局快速输入
- 模块 B：GTO 策略计算引擎
- 模块 C：策略结果展示
- 模块 D：历史记录与复盘
- 模块 E：设置与偏好

# 2 用户与场景

## 2.1 目标用户

| 用户类型 | 描述 | 核心痛点 |
| --- | --- | --- |
| 德州新手 | 入门 < 6 个月，了解基本规则但不懂 GTO | 不知道该 call/raise/fold，凭直觉行事 |
| 进阶玩家 | 有实战经验，了解 GTO 概念但缺工具 | 传统软件太复杂，无法快速查阅 |
| 线上多桌玩家 | 同时操作多桌，需要辅助决策 | 需要在 30 秒内给出行动建议 |

## 2.2 核心使用场景

**场景 A — 实时辅助（主场景）**
玩家在轮到自己行动前，用手机/平板快速输入牌面，获取 GTO 建议。目标操作时长 < 60 秒。

**场景 B — 赛后复盘**
牌局结束后回顾某一关键决策点，无时间限制，追求精确分析。

**场景 C — 主动学习**
玩家构造各类典型牌面情境，系统性学习 GTO 策略规律。

# 3 模块 A：牌局快速输入

## 3.1 综述

进入【DPoker — 主页面】，直接展示牌局输入区，无需任何注册/登录即可使用。

**用户角色及权限**
所有访客均可使用完整计算功能；登录用户额外享有历史记录保存功能。

**操作页面**
单页面三栏布局：左栏输入 → 中栏结果 → 右栏历史。移动端折叠为上下结构。

**核心操作流程**
```
① 选择街道 → ② 点选手牌 → ③ 点选公共牌 → ④ 设置牌局参数 → ⑤ 一键计算
```

**状态与操作映射**

| 当前状态 | 触发条件 | 可执行操作 |
| --- | --- | --- |
| 待输入 | 页面初始化 | 点选牌、设置参数 |
| 输入中 | 已选部分必填项 | 继续输入、清空重置 |
| 可计算 | 所有必填项已填写完整 | 点击「计算 GTO 策略」|
| 计算中 | 点击计算后 | 等待（展示 loading）|
| 已出结果 | 计算完成 | 查看结果、保存记录、重新输入 |

---

## 3.2 街道选择

页面顶部展示 4 个街道标签按钮，用户点击选择当前所处街道：

| 标签 | 显示文案 | 后续需输入公共牌数 |
| --- | --- | --- |
| Pre-Flop | 翻前 | 0 张 |
| Flop | 翻牌圈 | 3 张 |
| Turn | 转牌圈 | 4 张 |
| River | 河牌圈 | 5 张 |

选中标签高亮，公共牌输入区动态更新卡槽数量。

---

## 3.3 牌面可视化输入器

**设计核心：一个共用的 52 张牌格，点击选牌**

52 张牌按花色分 4 行、牌面值分 13 列展示（A K Q J T 9 8 7 6 5 4 3 2）。

**（1）选牌规则**

| 选牌槽位 | 选牌顺序 | 已用牌处理 |
| --- | --- | --- |
| 手牌 1、手牌 2 | 前两次点击 | 所选牌从牌格中置灰，不可再次选中 |
| 公共牌 1~5 | 后续点击（按街道数量限制） | 同上 |

用户可点击已选中的牌槽位「✕」按钮取消选择，对应牌格恢复可选。

**（2）手牌类型自动识别标签**
选完 2 张手牌后，自动展示：同花（Suited）/ 对子（Pair）/ 连牌（Connected）/ 同花连牌（Suited Connector）等标签，置灰不可编辑。

## 3.4 牌局参数设置

**（1）我的位置（Position）— 可视化牌桌点选**

展示 6 人桌示意图，用户点击自己所在位置：

```
         ┌──────────────┐
         │  BTN/Dealer  │
    CO ──┤              ├── SB
    HJ ──┤    牌 桌     ├── BB
         │              │
         └──── UTG ─────┘
               UTG+1
```

6 个位置按钮，选中后高亮。**位置是 GTO 策略中权重最高的变量，必填。**

**（2）有效筹码深度（Effective Stack）— 预设档位 + 滑块**

提供 5 个预设按钮（点击即选）：

| 预设 | 适用场景 |
| --- | --- |
| 20BB | 短筹码推推折 |
| 40BB | 中短筹码 |
| 60BB | 中等筹码 |
| 100BB | 标准深度（默认） |
| 150BB+ | 深筹码 |

滑块范围：10BB ~ 300BB，拖动可精细调整。

**（3）面对的行动（Facing Action）— 场景化快选**

根据当前街道，展示最常见行动选项（单选下拉）：

| 街道 | 快选选项 |
| --- | --- |
| Pre-Flop | 面对加注 2BB、2.5BB、3BB、面对 3-Bet、面对全押、主动发起 |
| Flop/Turn/River | 面对过牌、面对 1/3 池、面对 1/2 池、面对满池、面对超池下注、面对全押 |

**（4）底池大小（Pot Size）— 自动估算 + 手动覆盖**
系统根据街道、位置、面对行动自动估算底池（单位 BB），用户可通过滑块手动修正。

**（5）对手人数**
单选：1（单挑 Heads-Up） / 2 / 3+，默认为 1。

## 3.5 输入校验规则

| # | 校验项 | 校验逻辑 |
| --- | --- | --- |
| 1 | 手牌完整性 | 手牌 1 和手牌 2 均已选择，否则「计算」按钮置灰并提示"请选择您的手牌" |
| 2 | 公共牌完整性 | 非 Pre-Flop 街道下，公共牌数量需符合街道要求，否则提示"请补全公共牌" |
| 3 | 牌不重复 | 所选各牌不得重复，系统通过禁用已选牌格保证此规则，无需额外提示 |
| 4 | 位置已选 | 未选位置时，「计算」按钮置灰并提示"请选择您的位置" |
| 5 | 面对行动已选 | 未选面对行动时，「计算」按钮置灰 |

# 4 模块 B：GTO 策略计算引擎

## 4.1 计算架构概述

本工具采用**两层计算架构**：

```
第一层：预计算查找表（Lookup Table）
  ↓ 未命中
第二层：实时近似求解（Simplified Real-time Solver）
```

**第一层 — 预计算查找表**
离线预先计算覆盖高频牌局情境的 GTO 策略，存储为 JSON/Binary 格式。
- 覆盖：Pre-Flop 全部 169 种起手牌 × 6 个位置 × 常见面对行动 = 约 10,000+ 条目
- 查询耗时：< 50ms
- 存储体积：压缩后约 5MB（可客户端缓存）

**第二层 — 实时近似求解**
对查找表未覆盖的翻后街道情境，使用简化 CFR（Counterfactual Regret Minimization）算法实时求解。
- 简化假设：对手区间预设为标准范围（可由用户调整）
- 求解轮数：500~2000 次迭代（精度与速度平衡）
- 目标响应时间：< 3 秒

## 4.2 GTO 核心算法说明

**CFR 算法（反事实遗憾最小化）**

CFR 是目前德州扑克 GTO 求解的标准算法，通过反复自我博弈迭代，最终收敛到纳什均衡策略。

核心公式：
- 反事实遗憾：`R^T(I,a) = Σ (v(σ,I→a) - v(σ,I))`
- 策略更新：`σ^(T+1)(I,a) = R^T_+(I,a) / Σ R^T_+(I,a')`

**翻前策略简化**
翻前场景使用预计算的标准 GTO 开局范围（RFI Range）和应对范围（Calling/3-Betting Range），直接查表输出，无需实时求解。

**翻后权益计算**
使用蒙特卡洛模拟（10,000 次采样）计算当前手牌对抗对手预估区间的胜率（Equity）：
- 计算精度：误差 < ±2%
- 计算耗时：< 100ms（Web Worker 多线程）

## 4.3 对手区间预设

| 对手类型 | 区间描述 | 适用场景 |
| --- | --- | --- |
| 标准（默认） | GTO 标准区间 | 不了解对手时使用 |
| 宽松 | 含更多垃圾牌 | 面对鱼型玩家 |
| 紧张 | 只有强牌 | 面对保守型玩家 |
| 自定义 | 用户手动圈选区间图 | 有充分信息时使用 |

# 5 模块 C：策略结果展示

## 5.1 结果展示层级

计算完成后，结果区域从上至下展示以下内容：

**1 主行动建议（最高优先级）**

大字体彩色标签展示主建议：

| 建议 | 颜色 | 触发条件（GTO 频率） |
| --- | --- | --- |
| FOLD 弃牌 | #EF4444（红） | Fold 频率 ≥ 55% |
| CALL 跟注 | #F59E0B（黄） | Call 频率 ≥ 55% |
| RAISE 加注 | #10B981（绿） | Raise 频率 ≥ 55% |
| MIXED 混合 | #6366F1（紫） | 无单一行动 ≥ 55% |

**2 行动频率条形图**

水平堆叠条形图，分段展示各行动频率：
```
Fold  ████████░░  42%
Call  ████░░░░░░  28%
Raise ██████░░░░  30%  → 建议尺度: 2.5x 或 All-in
```

**3 权益展示（Equity）**

圆形进度图，展示当前手牌对抗对手区间的胜率：
```
        ╭──────╮
       │  63%  │  ← 你的权益
        ╰──────╯
   对抗标准区间的胜率
```

**4 新手解释卡片**

折叠展开的「为什么这样做？」卡片，2~3 句简明解释：
> 示例："你持有 A♠K♠（同花 AK），在 Q♥7♦2♣ 的干燥板面上有超对潜力和坚果后门同花听牌，GTO 建议以 30% 频率加注以保护区间价值，70% 跟注以控池。"

**5 进阶信息（可折叠）**

面向进阶用户展示：
- SPR（Stack-to-Pot Ratio）筹码底池比
- 最优下注尺度（Bet Sizing）列表
- 区间优势分析（Range Advantage）

# 6 模块 D：历史记录与复盘

## 6.1 综述

进入【DPoker — 历史记录】区域（主页面右侧栏或底部 Tab）。

**用户角色及权限**
未登录用户：历史记录存储于 localStorage，最多保存 20 条，清除浏览器数据后丢失。
登录用户：云端保存，无上限，支持搜索筛选。

**记录列表字段**

| 字段 | 说明 |
| --- | --- |
| 手牌 | 2 张手牌图标 |
| 街道 | Pre-Flop / Flop / Turn / River |
| 位置 | BTN / CO / HJ 等 |
| 建议行动 | Fold / Call / Raise / Mixed |
| 查询时间 | 精确到分钟 |

## 6.2 复盘功能

点击任意历史记录，可一键还原当时的牌局输入并重新查看结果，支持调整参数后对比不同情境的 GTO 建议。

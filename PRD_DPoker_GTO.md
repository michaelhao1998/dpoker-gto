# DPoker GTO 工具需求文档 V1.0
——德州扑克 GTO 快速决策计算器

---

# 1 整体说明

德州扑克玩家在实战中面临决策时间短、信息复杂的困境，传统 GTO 软件（PioSOLVER、GTO+）操作门槛极高，无法在牌桌旁快速使用。本产品旨在打造**面向新手友好、快速输入、即时输出 GTO 最优策略**的网页端工具，让玩家在 60 秒内完成完整策略查询。

**核心设计哲学：**
- 最小化输入：可视化交互替代文字输入，减少操作步骤
- 近似 GTO：预计算查找表 + 实时简化求解，平衡精度与速度
- 新手友好：策略输出附带简明解释

**该部分包括以下模块：**
1. 牌局信息快速输入模块
2. GTO 策略计算引擎
3. 策略结果展示模块
4. 历史记录与学习模块

# 3 功能模块

## 3.1 牌局信息输入模块

### 3.1.1 手牌输入

**设计原则：可视化点选替代文字键入**

展示 52 张牌的网格（4 行 × 13 列），按花色分行：♠♥♦♣。用户点击两张牌完成手牌选择，已选中的牌高亮，已被选作公共牌的牌自动置灰。

系统自动识别并展示手牌标签（suited / pair / connected 等）。

| 字段 | 格式 | 规则 |
| --- | --- | --- |
| 手牌1 | 牌面值+花色 | 必选，点选 |
| 手牌2 | 牌面值+花色 | 必选，点选 |
| 手牌类型标签 | 文字标签 | 置灰，系统自动识别 |

---

### 3.1.2 公共牌与街道

**街道选择器：** Pre-Flop → Flop → Turn → River 四段式标签切换

| 街道 | 公共牌数量 |
| --- | --- |
| Pre-Flop | 0 张 |
| Flop | 3 张 |
| Turn | 4 张 |
| River | 5 张 |

公共牌从同一牌格点选，已作为手牌的牌自动排除。

---

### 3.1.3 牌局参数（滑块 + 预设值替代手动输入）

**（1）位置选择（必填）**

可视化牌桌示意图，6 个位置按钮点击选中：UTG / UTG+1 / HJ / CO / BTN / SB / BB

**（2）有效筹码深度（必填）**

预设快选按钮：20BB / 40BB / 60BB / 100BB / 150BB+，支持滑块精确调整（10-300BB）

**（3）面对的行动（必填）**

| 街道 | 快选选项 |
| --- | --- |
| Pre-Flop | 面对加注 2.5BB、3BB、3-Bet、全押 |
| Post-Flop | 1/3 底池、1/2 底池、满池、超池、过牌 |

**（4）底池大小**
自动根据筹码深度和面对行动估算，支持手动调整（单位 BB）。

**（5）对手人数**
单选：1（单挑）/ 2 / 3+

| 字段 | 交互类型 | 是否必填 | 默认值 |
| --- | --- | --- | --- |
| 当前街道 | 4段选择器 | 必填 | Pre-Flop |
| 我的位置 | 可视化牌桌点选 | 必填 | 无默认 |
| 有效筹码深度 | 预设按钮/滑块 | 必填 | 100BB |
| 面对的行动 | 下拉快选 | 必填 | 无默认 |
| 底池大小 | 滑块（自动估算） | 必填 | 系统估算 |
| 对手人数 | 单选 | 必填 | 1 |

## 3.2 GTO 策略计算引擎

### 3.2.1 计算触发

所有必填项完成后，「计算 GTO 策略」按钮由置灰变为高亮可点击。

**计算流程：**
```
点击「计算」→ 前端参数校验 → 查询预计算缓存（< 200ms）→ 未命中则触发近似求解（< 3s）→ 返回策略
```

计算中显示加载动画，附文案「正在计算 GTO 最优策略...」

---

### 3.2.2 GTO 算法说明

**算法分层策略（精度由高到低）：**

**层级 1：预计算查找表（Lookup Table）**
- 针对最常见的 100-200 种翻前情境（位置 × 筹码深度 × 面对行动），离线预计算完整 GTO 解，存储为 JSON 查找表
- 命中率预期覆盖 80% 以上的查询请求
- 响应时间 < 100ms

**层级 2：近似 CFR 实时求解（Counterfactual Regret Minimization）**
- 对未命中查找表的情境，采用 Monte Carlo CFR（MCCFR）简化版本
- 输入参数：手牌区间（通过位置推断）、公共牌纹理、底池赔率
- 迭代次数：1000-5000 次（根据剩余时间动态调整）
- 使用 Web Worker 在后台线程计算，不阻塞 UI

**层级 3：启发式规则兜底（Heuristics）**
- 计算超时（>5s）或异常情况下，使用基于数学期望的简化规则给出建议
- 明确标注为「简化策略」，提示用户精度较低

**核心计算参数：**

| 参数 | 说明 | 计算方式 |
| --- | --- | --- |
| 手牌权益（Equity） | 当前手牌对抗对手区间的胜率 | Monte Carlo 模拟 10,000 次 |
| 底池赔率（Pot Odds） | 跟注成本 / 底池总额 | 直接计算 |
| 期望价值（EV） | 各行动的预期收益 | CFR 迭代输出 |
| 区间优势（Range Advantage） | 我方区间 vs 对手区间的结构优势 | 基于位置和板面的近似计算 |

### 3.3.1 主行动建议

---

### 3.3.4 新手解释卡片（重要特色功能）

折叠展开式「为什么这样做？」卡片，以 2-3 句话解释策略逻辑：

> 示例："你持有 A♠K♠，在此 A♠7♥2♣ 翻牌面上你有顶对顶踢脚（TPTK）。GTO 建议 70% 下注是因为你的区间在此板面有显著优势，用 1/3 底池小注可以提取价值同时保护你的区间平衡。"

解释文案生成规则：
- 手牌强度（强牌/中等牌/弱牌/听牌）→ 触发对应文案模板
- 板面类型（干燥板/湿润板/配对板）→ 调整解释逻辑
- 位置优势/劣势 → 补充位置说明

---

### 3.3.5 策略可信度标识

| 标识 | 说明 |
| --- | --- |
| ★★★ 高精度 | 来自预计算查找表 |
| ★★☆ 中精度 | 来自 MCCFR 实时计算（≥3000次迭代）|
| ★☆☆ 近似值 | 来自启发式规则，仅供参考 |

# 4 输入简化设计规范（UX 核心）

## 4.1 步骤简化对比

| 传统工具操作步骤 | DPoker 简化方案 |
| --- | --- |
| 手动输入手牌（如 "AsKs"） | 点击牌格选 2 张牌 |
| 设置完整区间（数百次操作） | 省略：用位置自动推断标准区间 |
| 设置树结构（下注尺度等） | 省略：使用预设标准下注尺度 |
| 手动输入底池/有效筹码 | 滑块预设值，自动估算 |
| 等待 30s-5min 求解 | < 3s（查找表 + 近似算法）|
| 理解复杂的策略树 | 一键输出主行动 + 新手解释 |

**估算：传统工具需要 20-50 步操作，DPoker 目标 8-10 步完成输入**

## 4.2 输入步骤设计

```
Step 1：点选手牌（2次点击）
Step 2：选择街道（1次点击）
Step 3：点选公共牌（0-5次点击，翻前可跳过）
Step 4：选择位置（1次点击）
Step 5：选择有效筹码预设（1次点击，默认100BB可直接跳过）
Step 6：选择面对的行动（1次点击）
Step 7：点击「计算」（1次点击）
——————————
合计：7-13 次点击，无需键盘输入
```

## 4.3 智能默认值与省略策略

| 省略/简化项 | 省略理由 | 处理方式 |
| --- | --- | --- |
| 对手具体区间 | 新手无法准确评估，且精度影响有限 | 根据位置使用标准默认区间 |
| 完整下注尺度树 | 增加复杂度，收益低 | 提供 3 种标准尺度：1/3 / 1/2 / 满池 |
| 多街道历史行动 | 输入成本高 | 仅基于当前街道计算，近似处理 |
| 精确底池大小 | 可接受误差范围 ±20% | 自动估算，支持手动微调 |
